# Generated TypeScript bindings

These files are **auto-generated by [ts-rs](https://github.com/Aleph-Alpha/ts-rs)** from the Rust source of truth in the backend structs that derive `TS`. **Do not edit by hand.**

## Regenerate

```sh
npm run gen:bindings
```

(Equivalent to `cd src-tauri && cargo test --lib export`. The `#[ts(export)]` attribute on each type emits a synthetic test that, when run, writes the binding into this directory. The destination is set by `TS_RS_EXPORT_DIR` in `src-tauri/.cargo/config.toml`.)

## When to regenerate

- After adding/renaming/removing a field on any type that derives `TS`.
- After adding `#[derive(TS)]` to a new type.
- Before committing — CI runs `npm run gen:bindings` and fails if the diff is non-empty.

## What's covered

The current closure covers:

- The BE→FE event surface: `AssistantUiEvent`, `AssistantEventEnvelope`, and every type they transitively reach (sessions, messages, runs, tool invocations, ask_user options, run notices, …).
- The workspace surfaces the FE reads directly: `WorkspaceSnapshot`, `WorkspaceListEntry`, workspace file/session response types, and their nested task/agent/file entries.

`ExecutionCapabilityConfig` is intentionally erased to `unknown` in these bindings until the settings/provider surfaces are migrated.

## What's NOT covered

- The remaining Tauri command request/response shapes. Add `#[derive(TS)]` on a command's request struct and return type when typing a specific command boundary.
