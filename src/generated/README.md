# Generated TypeScript bindings

These files are **auto-generated by [ts-rs](https://github.com/Aleph-Alpha/ts-rs)** from the Rust source of truth in `src-tauri/src/assistant/{types,events}.rs` and `src-tauri/src/assistant/tools/ask_user.rs`. **Do not edit by hand.**

## Regenerate

```sh
npm run gen:bindings
```

(Equivalent to `cd src-tauri && cargo test --lib export`. The `#[ts(export)]` attribute on each type emits a synthetic test that, when run, writes the binding into this directory. The destination is set by `TS_RS_EXPORT_DIR` in `src-tauri/.cargo/config.toml`.)

## When to regenerate

- After adding/renaming/removing a field on any type that derives `TS`.
- After adding `#[derive(TS)]` to a new type.
- Before committing — CI / a pre-commit hook should also run `npm run gen:bindings` and fail if the diff is non-empty (TODO).

## What's covered

The current closure is the BE→FE event surface: `AssistantUiEvent`, `AssistantEventEnvelope`, and every type they transitively reach (sessions, messages, runs, tool invocations, ask_user options, run notices, …). `ExecutionCapabilityConfig` is intentionally erased to `unknown` because the FE doesn't deeply consume the execution capability tree in events — narrowing the bindings to what the FE actually uses.

## What's NOT covered

- Tauri command request/response shapes (the `invoke('cmd_name', { ... })` payloads). Add `#[derive(TS)]` on the command's request struct + the return type when typing a specific command boundary.
- `WorkspaceSnapshot` and the workspace-page contract — separate surface; add when needed.
